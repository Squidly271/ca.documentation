Community Applications (Apps Tab) Module

Overview

 Community Applications is a plugin designed to make finding, installing, and managing docker and plugin applications. When using this plugin, there is no longer any need to use the Repositories section of the Docker Tab From top to bottom will explain all aspects of Community Applications starting from the basic and going to the more advanced options. All items within this plugin have tooltips enabled to further guide you in operation. For a help for the settings for this plugin, load the settings page and press Help. Each item will have associated help text displayed. After installation, a new tab will appear on unRaid's main menu called "Apps". Additionally, there will be a section within the Settings tab called Community Applications which has some extra utilities to customize the operation of this plugin.
To begin with, at the top of the page are the navigation buttons. Clicking or hovering over these buttons will change the displayed results accordingly:
- This chooses which "section" to display (Available Apps, Installed Apps, Previously Installed Apps, Pinned Apps, additional CA Modules
- This chooses how to display the results
- This selects a category of apps to display
- When available, this selects a subcategory of apps to display

Displayed Results

 After selecting a category (or searching),the applications will appear: (example from Icon Details Mode)



Clicking or hovering over the icon will bring up more details about the application.



The following action icons may be available for any given application:

- These icons respectively will Install, Edit, or Update an application
- Clicking this icon will take you to an application's GUI

Searching
At the top right of the page is a search box. Enter a search string here to search for either an application's name, author's name, repository name, or description



Installed / Previously Installed Applications
Selecting Installed Apps from the "Section" icon will display all applications which are currently installed, and allow you to edit them.
A will appear next to the application. Selecting this will allow you to uninstall the application, and optionally delete the appdata folder for the application (See Settings section)


Selecting Previous Apps from the "Section" icon will display all applications which have been previously installed, and allow you to re-install them.
A will appear next to the application. Selecting this will remove (ie: delete the user template) from the list of previously installed applications

Pinning Applications
On any of the displays (with the exception of when dockerHub search results are shown), you can "pin" an application for later viewing. This is useful for when you're just skimming through the lists and come across an application that you'd like to try but don't have the time to do it right away.

Pressing the will pin the application whereas pressing the will unpin the application.

Pinned applications can be viewed by selecting Pinned Apps from the "Section" Icon



Advanced - dockerHub Integration

There is a setting which you can enable which will allow you to search dockerHub for other applications beyond the ones which have had unRaid templates created for them.
When this option is enabled, a few more options are available.
  • Clicking on the Application's Name of any application will then search dockerHub for that application.
  • After performing any search for applications, a button will appear () to then allow you to search dockerHub for that application.
    Within the dockerHub search results, you can navigate and search as before. Clicking the author's name will search for all applications from that author. Clicking the application's name will search for more results for that application name.
    Clicking will attempt to create a template for that application and install it. There are however some caveats with regards to the conversion process. See this thread for more details: http://lime-technology.com/forum/index.php?topic=40262.msg401570#msg401570
    At a later time if you wish to manage the application added (Edit / restore to defaults, etc), the application will be available under the UNCATEGORIZED category. (Alternatively, you could categorize the plugin by using the Docker Categorizer plugin HERE
    Depending upon what search term is currently being used, Community Applications may also offer you some alternate search terms for more relevant containers.
    NOTE: When adding an application from dockerHub, popups must be enabled within your browser

    Private Repositories
    If you have access to a so-called private repository (or wish to create your own), there are two options for integrating it into CA
    Create a folder called /boot/config/plugins/community.applications/repositoryName

    Within this folder, place any custom / private templates you might have. (If using the dockerHub conversion section of this plugin, a folder will be automatically created here called dockerHub


    The templates contained in the private repository will be flagged when displayed as Private

    General Settings
    Within the apps tab, there is a link called "CA Settings" (top right). Clicking this link will allow you to fine tune the operation of the apps tab

    Time to display applications as new / updated
    This option dictates the amount of time that a new or updated application will be flagged as new and displayed under the New / Updated category. Defaults to 3 Months
    Favourite Repository
    This option allows you to set a "favourite" repository. If set, then applications from that repository will display before all others. Defaults to None
    Show already installed apps within available apps
    By default, if an application has been installed, it will not appear within the available apps (It will only show within Installed Apps). Selected this will display the installed apps within the available apps lists
    Hide Deprecated Applications
    By default, CA will not display any deprecated applications within the available apps. It will always however show you any apps that have been deprecated within the Installed or Previous Apps sections. As a general rule, it is not recommended to install deprecated applications onto your server Hide Incompatible Applications
    If CA thinks that an application is incompatible with the version of unRaid that you are using, by default it will not display the application.
    Display Random App Of The Day
    Enables/disables the displaying of random apps of the day when entering the Apps tab
    Search bar is always in focus
    If the search bar is always in focus, at any point you can type in a search term and CA will search for it. If the search bar is not always in focus, you will first have to click into the search bar to type.
    Setting this to "Automatic" enables this feature on computers, and disables it on mobile devices.
    Enable additional search results from dockerHub
    This option enables the plugin to perform additional searches from dockerHub when looking for an application. See above (Advanced - dockerHub integration for more details) Default: No
    Other Notes

    Community Applications (and its predecessor Community Repositories) introduced a number of additional XML tags in order to perform it's magic. Any application / template author and/or maintainer should check out the list of schema additions here: http://lime-technology.com/forum/index.php?topic=40299.0
    Depending upon which browser you are using, there may be a slight delay (~1-2 seconds) in displaying the results (in particular, All Applications using Table mode). Some browsers are faster than others are sorting out all the displayed information. In my testing, on a PC, Chrome is by far the fastest at rendering the results. Note that generally whenever CA searches or re-renders the display, a spinning ring will appear. When that ring stops spinning, CA has finished, and any delays from that point onwards are 100% entirely due to your browser
